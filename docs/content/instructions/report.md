## Оформление

**В репозиторий положить файл Readme.md, в котором описаны инструкции**

<div class="small-table" style="margin-top: 20px">

| # | **Требование**                                                              | **Статус** | **Комментарий**                                                                              |
|---|-----------------------------------------------------------------------------|------------|----------------------------------------------------------------------------------------------|
| 1 | В репозитории есть `README.md`. Но лучше пользоваться порталом документации | ✔️         | Файл `README.md` добавлен в корень репозитория                                               |
| 2 | Описаны инструкции по развёртыванию приложения                              | ✔️         | Содержит шаги запуска backend и frontend (локально и в k8s)                                  |
| 3 | Описано устройство репозитория                                              | ✔️         | Описана структура каталогов: `backend`, `frontend`, `.lib`, `charts`, `terraform`, `ansible` |
| 4 | Описано развёртывание инфраструктуры                                        | ✔️         | Приведены команды `terraform apply` и `ansible-playbook`, требования к окружению             |
| 5 | Описаны правила внесения изменений в инфраструктуру                         | ✔️         | Описан git-flow для infra, порядок изменения Terraform/Ansible, проверки перед merge         |
| 6 | Описан релизный цикл приложения и правила версионирования                   | ✔️         | Описано использование `.version`, теги образов, ветки `develop`/`main`, правила релизов      |
